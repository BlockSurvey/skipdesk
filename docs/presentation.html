05
Tech approach — models, data, architecture

Edge-native. Multi-tenant. One data layer.

  • Models: Vapi for real-time voice (shared assistant, multi-tenant at the edges); Claude for reasoning & tool use; Workers AI (bge-base-en-v1.5) for embeddings + doc conversion.
  • Data: Cloudflare D1 (SQLite/Drizzle, one source-of-truth schema); R2 for doc blobs; knowledge-base RAG grounds answers in the business's own content.
  • Interface: an MCP server (16 tools) and REST — both call the identical tenant-scoped queries. No duplicated logic.
  • Tenancy: every query scoped by business_id from the authenticated principal — never caller input. No cross-tenant read path.
  • Auth: PBKDF2 passwords + stateless ES256 JWT (worker signs, UI verifies locally).
